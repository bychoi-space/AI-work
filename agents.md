# LF Editor Studio를 위한 Antigravity 에이전트 제약 조건

## 🛠️ 기술 스택 및 아키텍처 (엄격한 규칙)
- **Vanilla JS 전용**: 프레임워크(React, Vue 등)를 절대 사용하지 마세요. 코드는 가볍고 직관적으로 유지해야 합니다.
- **단일 진실 공급원 (SSOT)**: 모든 프로젝트 상태(화면 목록, 순서, 설명 등)는 각 프로젝트 폴더의 `metadata.json`에서만 배타적으로 관리되어야 합니다.
- **주석 (Annotations)**: 반드시 **Quill.js**를 사용하세요. 스타일 보존을 위해 주석 데이터는 `metadata.json`에 HTML 형식으로 저장해야 합니다.
- **상호작용 분리**: 마커 드래그(위치 이동)와 클릭(편집 모드 진입)은 논리적으로 완벽히 분리되어야 합니다.

## 🎨 UI 및 컴포넌트 규칙
- **캔버스 크기**: 에디터 화면(`lf-canvas` 또는 `page`)의 크기는 레이아웃 틀어짐을 방지하기 위해 'Cover' 화면 크기(예: 1440x900)와 완벽하게 일치해야 합니다.
- **타이포그래피 가이드 (폰트 크기 표준)**: 가독성 확보를 위해 다음 기준을 따르세요.
  - 대분류 타이틀 (화면 제목 등): `18px` ~ `20px`
  - 중분류 헤더 (테이블 헤더, 그룹 제목): `15px` ~ `16px`
  - 본문 텍스트 (테이블 셀, 일반 텍스트): `14px` ~ `15px`
  - 부가 설명 및 라벨 (서브헤더, 개별 라벨, 주의사항): `13px`
  - 최소 표기 단위 (매우 작은 마커/태그 텍스트): `12px`
- **텍스트 디테일 및 정렬 (레이아웃 보호)**: 
  - 한 줄로 유지되어야 하는 텍스트(예: 날짜, 짧은 라벨)는 반드시 `white-space: nowrap;`을 적용하여 텍스트 줄바꿈(글자 잘림)을 방지하세요.
  - CSS Polygon 등의 도형 안에 텍스트를 넣을 때는 도형의 여백(Padding)과 줄간격(`line-height`)을 세밀하게 계산하여 텍스트가 도형의 한가운데에 완벽하게 정렬되도록(어색하게 쏠리지 않도록) 구성하세요.
- **컴포넌트 시스템**: 드래그 앤 드롭을 지원하기 위해 편집 가능한 모든 주요 요소는 `.lf-component`로 감싸야 합니다.
- **버튼 디자인 표준 (Premium UI)**: 모든 버튼은 [JIRA 바로가기] 버튼의 콤팩트하고 정밀한 스타일을 절대적으로 따릅니다.
  - **쉐입 (Shape)**: 양쪽 끝이 완전히 둥근 **알약형(Pill) 쉐입**을 적용합니다. (`height: 26px`, `border-radius: 13px`)
  - **컬러 체계 (Hierarchy)**:
    - **Accent (`btn-accent`)**: 가장 핵심적인 버튼 (강조컬러 + 흰색텍스트 + 얇은 테두리)
    - **Primary (`btn-primary`)**: 상대적으로 중요한 버튼 (검정컬러 + 흰색텍스트 + 얇은 테두리)
    - **Secondary (`btn-secondary`)**: 일반 버튼 (밝은컬러 + 검정색텍스트)
  - **시인성 확보**: 어두운 계열의 버튼(`btn-accent`, `btn-primary`)은 배경색과 겹치지 않도록 반드시 미세한 밝은색 테두리(`rgba(255, 255, 255, 0.15~0.25)`)를 적용합니다.
  - **아이콘 규칙**: **핵심/중요 버튼**에만 아이콘을 적용하며, 일반 버튼에는 아이콘을 사용하지 않습니다. (굵기 `Regular/Outlined`, 크기 `13px`)
  - **텍스트**: 폰트 크기 `11px`, 굵기 `500` (Medium).
  - **상호작용**: 모든 버튼은 호버 시 입체적으로 반응(`translateY(-2px)` + 그림자 강화)하도록 통일합니다.
  - **정렬**: 좌우 여백(`16px`)을 동일하게 처리하고, 아이콘과 텍스트는 버튼의 정중앙에 완벽히 정렬합니다.
- **모달 및 팝업**: 시스템 내의 모든 팝업(삭제, 경고, 수정 등) 버튼도 위 디자인 표준을 100% 동일하게 적용해야 합니다.
- **편집 영역**: 에디터 인스펙터와 연동되도록 텍스트 영역에는 `contenteditable="true"` 속성과 `.v4-editable-cell` 클래스를 사용하세요.
- **삭제 트리거**: 사용자가 오브젝트를 삭제할 수 있도록 항상 `.lf-delete-trigger` (`×` 버튼)를 주입하세요.

## 🛡️ 보안 및 통신 규칙 (매우 중요)
- **파일 프로토콜 준수**: `file://` 환경에서는 iframe에 대한 직접적인 DOM 접근(`contentDocument`)이 차단됩니다. 교차 창 간의 모든 비동기 통신에는 반드시 **`postMessage`** API를 사용해야 합니다.
- **의존성 인라인화**: `srcdoc` iframe에서의 보안 차단을 방지하기 위해, 핵심 CSS/JS 의존성은 HTML 문자열 내에 직접 인라인으로 주입하세요.

## ⚠️ 에이전트의 흔한 실수 방지
- **정밀한 분석 후 진행**: 작업을 시작하기 전, 사용자 요구사항과 기존 코드를 픽셀 단위까지 세세하게 분석하고 확인하여 '단 한 번에 확실하게' 진행하세요. 똑같은 이야기를 반복하게 만드는 것은 용납되지 않습니다.
- **사이드 이펙트 방지 (회귀 방지)**: 특정 기능을 수정할 때 다른 멀쩡한 코드(예: 이벤트 리스너, 타 컴포넌트 스타일)를 망가뜨리지 않도록 전체적인 영향도를 반드시 검토하세요.
- **중복 리스너 방지**: `init()` 함수는 파일당 오직 **한 번만** 정의하세요. 여러 번 정의하면 기존 리스너를 조용히 덮어쓰게 됩니다.
- **반응 없는 UI 방지**: `init` 단계에서 모든 UI 요소에 이벤트 리스너를 명시적으로 연결하세요.
- **병합 시 데이터 유실 방지**: 충돌 발생 시 절대 `metadata.json`을 맹목적으로 덮어쓰지 마세요. 반드시 `screens` 배열을 수동으로 병합(Merge)해야 합니다.


## 🔀 프로세스(플로우차트) 화면 제작 규칙
- **절대 좌표 및 스크롤 방지**: 노드와 연결선은 1440x900 캔버스 내에서 `position: absolute`로 배치하며, 브라우저 스크롤이 발생하지 않도록 레이아웃을 최적화해야 합니다.
- **논리적 분기 및 조건 명시**: 다이아몬드(Diamond) 형태의 조건 분기점(예: 승인 여부, 가용화 등)에는 반드시 'Y'와 'N' 텍스트 노드를 배치하여 조건부 흐름의 방향을 명확히 정의해야 합니다.
- **레이어 및 흐름 구분**: 프로세스의 주체나 시스템(예: LFMall, OMS/SAP, POS 등)이 변경될 경우 가로 점선(Divider)을 사용하여 시각적 구역을 명확히 분리해야 합니다.
- **정밀한 노드 연결**: 직각 화살표(`v4-shape-arrow-elbow`)와 직선 화살표를 혼합하여 사용할 때, 노드와 선이 어긋나지 않도록 픽셀 단위로 정밀하게 위치를 조정해야 합니다.

## 🚀 버전 관리 및 배포 권한
- **작업 전 동기화**: `main` 브랜치와 동기화하기 위해 작업을 시작하기 전에 항상 `git fetch` 및 `git pull`을 실행하세요.
- **직접 푸시 (Direct Push)**: Antigravity AI는 로컬 검증 후 사용자의 확인 없이 `main` 브랜치에 `git add`, `commit`, `push`를 수행할 수 있는 명시적인 권한을 갖습니다. (단, 삭제와 같은 파괴적인 작업은 예외입니다).
